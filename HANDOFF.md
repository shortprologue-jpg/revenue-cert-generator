## 🔴 발견 1 — 노션에서 "목차"만 읽고 있다 (최우선 수정 대상)

`src/notion_fetcher.py`의 `fetch_all_blocks()`는 **최상위 블록만** 읽고
`has_children`(토글) 안으로 **재귀하지 않는다.**

권혁진님 성장 히스토리 페이지 실측:
- 최상위 블록 **572개** → 뽑히는 텍스트 **9,185자** = 전부 **토글 제목뿐**
- 자식 가진 블록 **168개**(toggle 218개)의 **내용이 통째로 누락**

즉 현재 뽑히는 건 `과제체크...`, `콘관시 체크`, `콘텐츠 피드백...` 같은 **라벨뿐**이고,
`prompts/system_prompt.md:31`이 요구하는 실제 내용(과제체크·콘텐츠 피드백·주차별 트레이닝·과제)은
전부 토글 **안**에 있어 반영되지 않는다.
→ **인증글이 사실상 카톡 내용만으로 작성되고 있다.** 노션 연결이 무의미한 상태.

## 🔴 발견 2 — 타임아웃 원인

토글 **3개** 여는 데 **6.1초** 실측. 168개 전부 열면 **5분 초과** → 사용자가 겪은 멈춤의 원인.
따라서 "토글을 열되 **필요한 주차만**" 열어야 한다. 무작정 재귀하면 더 심해진다.

## 🔴 발견 3 — 읽기 범위 로직이 잘못돼 있다

- **현재 코드**: `compute_cutoff_date()` = `period_end - 7일` → 그 **이전 전부**를 포함.
  - 기간이 6/1~6/30이면 → **1주차부터 6/23까지** 다 들어가고, **6/23~6/30은 오히려 빠진다.**
- **사용자가 원하는 동작 (확정)**: **[기간 시작 - 약 7일] ~ [기간 끝]** 구간만.
  - 예: 기간 6/1~6/30 → **5/25 ~ 6/30** 주차만 읽기.
  - 시작 1주 전을 넣는 이유 = 인증글의 "비포(원래 문제)" 근거 확보.
  - ⚠️ **회원마다 트레이닝 요일이 달라** 정확히 -7일 지점에 기록이 없을 수 있음.
    날짜를 칼같이 자르지 말고 그 근처 주차부터 자연스럽게 잡을 것. "무조건 -7일" 아님.
  - → `compute_cutoff_date` / `filter_blocks_by_cutoff` **재설계 필요**.

## ⚠️ 발견 4 — 마스터 DB는 이 앱 열쇠로 접근 불가

- 회원일지 프로젝트의 마스터 DB(`1a31a6604ea0808199e1ef2cb275368e`)는
  이 앱 통합(**`BPT 수익화 인증 자동화`**)에 **공유돼 있지 않다** → `ObjectNotFound`.
- **하지만 우회 불필요**: `client.search()`로 **성장 히스토리 페이지가 직접 보인다.**
  마스터 DB → 회원 페이지 → 성장 히스토리 3단계를 거칠 필요 없이 **바로 접근 가능**(더 단순·빠름).
- 현재 **14명**만 보임 (계획서상 회원은 28명):
  `고미선 권미혜 권혁진 김은수 박대현 양지원 오상엽 윤용남 이도하 이정헌 이찬희 이택민 조성환` + 무명 1개
- **사용자 액션 대기 중**: 빠진 회원의 성장 히스토리 페이지(또는 공통 부모 페이지)에
  `···` → `연결` → `BPT 수익화 인증 자동화` 추가 필요. 부모에 걸면 하위 일괄 적용될 수 있음.

## 다음 할 일 (우선순위대로)

1. **회원 드롭다운 구현** — 사용자가 드롭다운 방식으로 확정함(이름 타이핑보다 오타·동명이인에 강함).
   - `client.search()` → 제목에 "성장 히스토리" 포함 페이지 수집 → `^(.+?)님\s*성장 히스토리` 로 회원명 추출.
   - Streamlit `selectbox`는 타이핑 필터 내장 → "이름 치면 나오는" 요구 충족.
   - 결과: 입력칸 5개 → 3개. **노션 URL 입력·회원 이름 입력이 사라진다.**
   - 명단 캐싱 권장(`st.cache_data`) — 매번 search 하면 느림.
2. **읽기 범위 재설계** (발견 3) — `[기간시작-7일] ~ [기간끝]` 주차만 슬라이스.
   - 주차 경계 = `heading_2`, 제목에 `YYMMDD` 포함 (예: `🟠260717/ 28주차 / ...`). 최신이 맨 위.
   - 권혁진님 페이지 기준 `heading_2` 34개 = 34주차.
3. **선택된 주차 구간만 토글 재귀 열기** (발견 1+2 동시 해결).
   - 참고 구현: `bpt-member-log`의 `fetch_week.py` `Collector.render()` (depth 제한 + 블록 상한).
   - 속도 상한 필요(회원일지는 `MAX_BLOCKS=1200`, `MAX_CHILD_DEPTH=4` 사용).
4. 실제 회원으로 인증글 생성 E2E 검증 → 노션 내용이 실제로 반영되는지 확인.

## 주요 결정
- 저장소 이름은 영문 `revenue-cert-generator`(URL·클론 편의).
- 브랜치는 `main` 하나만 사용(혼자 두 노트북 오가는 구조).
- 비밀키는 절대 커밋하지 않음 — `.env`는 각 노트북 로컬 전용. (`.gitignore` 2번 줄에서 차단 확인)
- **`bpt-member-log`와 코드를 섞지 않는다.** 구조만 참고하고 이 프로젝트에 맞게 새로 작성.
- 회원 선택은 **드롭다운**(마스터 DB 아님, `search` 기반).
- `.bat`은 특정 노트북 경로를 하드코딩하지 않는다(두 노트북 공용).
